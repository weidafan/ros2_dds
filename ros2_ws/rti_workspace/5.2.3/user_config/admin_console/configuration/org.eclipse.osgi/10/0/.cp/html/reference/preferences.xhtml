Preferences
Overview
The preferences dialog contains controls to customize Admin Console to your needs and/or preferences. There are many options which come from the Eclipse platform and which will not be covered in detail here. Instead, this documentation will focus on those settings that are specific to Admin Console.

Administration
The Administration preference page controls how Admin Console interacts with DDS. Here you can set the cleanup period for entities which are no longer present () You can also choose to automatically join discovered active domains or manually join them. There is also a display of the currently joined domains (where you can choose to leave domains if you're manually joining domains). Finally, you can add QoS files from which you can choose profiles that Admin Console will use when creating DDS entities. Please take care when changing the profile that Admin Console uses since it employs many non-standard QoS settings. Start with the example QoS file in <installation dir>/resource/xml/admin_console_example_profiles.xml.

Health
The Health preference page controls the severity of each health reason. They are:
  • Error log entry - The severity of an error log message (received from Distributed Logger).
  • Warning log entry - The severity of a warning log message (received from Distributed Logger).
  • Requested/offered QoS - The severity of a mismatched DDS Quality of Service.
  • Transport message size max - The severity of mismatched DomainParticipant transport message size max settings.
  • Transport identifiers - The severity of mismatched DomainParticipant transport identifiers.
  • No matching partitions - The severity of not having matching partitions.
  • Keyed consistency - The severity of a disagreement on whether the topic is keyed or not.
  • Data type mismatch - The severity of having incompatible data types.
  • Writer-only Topic - The severity of having only DataWriters (and hence, no flow of data) associated with a topic.
  • Reader-only Topic - The severity of having only DataReaders (and hence, no flow of data) associated with a topic.

Logging
The Logging preference page controls RTI Logger configuration and verbosity settings used by Admin Console. The output of the RTI Logger is displayed in the Console Log View.

Notifications
The Notifications preference page controls if guide notifications (messages that replace a view's content with helpful hints about why the view might be empty) are displayed and whether a notification is displayed for successful configuration of Routing Service.
