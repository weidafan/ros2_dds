Domain Setup

RTI Admin Console uses Connext to communicate with other RTI services and Connext processes. Admin Console is configured by default to automatically discover and join all active domains in your distributed system. If you wish to manually control the joining and leaving of domains, then open the Preferences dialog with this button or from the menu, select View -> Preferences.


RTI services Setup

To use RTI Admin Console with other RTI services, administration, distributed logging, and monitoring (if supported) should be enabled. The following snippet shows an example of how to setup RTI Routing Service for use with Admin Console.
  • RTI Routing Service: 
    	  
<routing_service name="default">
	<annotation>
		<documentation>Description of your Routing Service</documentation>
	</annotation>		

	<!-- The administration tag below enables Administration and Distributed
	     Logger on domain 0, and is required when using Admin Console. -->
	<administration>
		<domain_id>0</domain_id>
		<distributed_logger>
			<enabled>true</enabled>
			<filter_level>WARNING</filter_level>
		</distributed_logger>
	</administration>
	
	<!-- The monitoring tag enables Monitoring configuration for all the entities in  
             this routing service, and are required when using Admin Console. -->
	<monitoring>
		<!-- The domain id where to publish the monitoring information -->
		<domain_id>0</domain_id>
		<!-- How often to publish it -->
		<status_publication_period>
			<sec>5</sec>
			<nanosec>0</nanosec>				
		</status_publication_period>
		<!-- How often to compute statistics -->
		<statistics_sampling_period>
			<sec>1</sec>
			<nanosec>0</nanosec>
		</statistics_sampling_period>
		<participant_qos>
			<participant_name>
				<name>RoutingService</name>
			</participant_name>
		</participant_qos>
	</monitoring>
	
	
	    ...

</routing_service>
For other services, see the User's Manual of each service for more details.

Important Note: For Routing & Replay Service, do not use the "name" tag inside the administration tag. This is a known issue and will be resolved in a future release.


Troubleshooting Admin Console
In cases where troubleshooting Admin Console is necessary, you can view the logs for Admin Console by selecting the menu View -> Show Console Log.
By default, the full log file is in "<My Documents>/rti_workspace/<product version>/user_config/admin_console /admin_console.log" on Windows systems, and in "~/rti_workspace/<product version>/user_config/admin_console/admin_console.log" on Linux & Mac systems. For other known issues with Admin Console, see the Release Notes document.
