Detailed Description

Information about the status dds::core::status::StatusMask::liveliness_changed()

The dds::sub::DataReaderListener::on_liveliness_changed callback may be invoked for the following reasons:

  • Liveliness is truly lost - a sample has not been received within the timeframe specified in dds::core::policy::Liveliness::lease_duration

  • Liveliness is recovered after being lost.

  • A new matching entity has been discovered.

  • A QoS has changed such that a pair of matching entities are no longer matching (such as a change to the dds::core::policy::Partition). In this case, RTI Connext will no longer keep track of the entities' liveliness. Furthermore:
    • If liveliness was maintained: dds::core::status::LivelinessChangedStatus::alive_count will decrease and dds::core::status::LivelinessChangedStatus::not_alive_count will remain the same.
    • If liveliness had been lost: dds::core::status::LivelinessChangedStatus::alive_count will remain the same and dds::core::status::LivelinessChangedStatus::not_alive_count will decrease.
