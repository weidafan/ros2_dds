Detailed Description

DDS_LIVELINESS_CHANGED_STATUS

The DDSDataReaderListener::on_liveliness_changed callback may be invoked for the following reasons:

  • Liveliness is truly lost - a sample has not been received within the timeframe specified in DDS_LivelinessQosPolicy::lease_duration

  • Liveliness is recovered after being lost.

  • A new matching entity has been discovered.

  • A QoS has changed such that a pair of matching entities are no longer matching (such as a change to the DDS_PartitionQosPolicy). In this case, RTI Connext will no longer keep track of the entities' liveliness. Furthermore:
    • If liveliness was maintained: DDS_LivelinessChangedStatus::alive_count will decrease and DDS_LivelinessChangedStatus::not_alive_count will remain the same.
    • If liveliness had been lost: DDS_LivelinessChangedStatus::alive_count will remain the same and DDS_LivelinessChangedStatus::not_alive_count will decrease.
Examples:
HelloWorld_subscriber.cxx.
